# Contribution

Merci de votre intérêt pour contribuer à `CharleBin` ! Ce document décrit les règles et le flux de contribution.

## Règles générales
- Lisez le `README.md` et les tests existants avant toute contribution.

## Flux de travail (branches et PR)
1. Cloner le dépot
2. Créez une branche dédiée 
3. Implémentez vos changements sur la branche.
4. Faites des commits 
5. Poussez votre branche 
6. Ouvrez une Pull Request (PR) vers la branche `main` du dépôt principal en :
   - Ajoutant un titre clair.
   - Rédigeant une description qui explique le pourquoi et le quoi.
   - Liant l'issue associée si applicable.

## Modèle rapide de PR (à coller dans la description)
- **Contexte** : court résumé du problème.
- **Changements** : ce qui a été modifié.
- **Tests** : comment tester localement.
- **Checklist** :
  - [ ] Tests unitaires ajoutés/validés
  - [ ] Linter passé
  - [ ] Documentation mise à jour si nécessaire

## Conventions de commits
- Utilisez des messages clairs et impératifs, exemple : `Fix: corrige l'erreur X` ou `Feat: ajoute la validation Y`.
